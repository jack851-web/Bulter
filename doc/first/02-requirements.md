# Bulter 详细需求文档

> 按"模块 → 功能条目"组织，每条需求独立可验收。

---

## 一、输入路径

### 路径 A：浮窗截图

1. 系统级 Overlay 浮窗，任意 App 长按触发截图
2. 截图后 AI 自动推理场景（聊天/账单/文章/体检报告等）
3. 推理结果归类到对应模块，支持用户确认/修改
4. 支持多张截图批量处理
5. 截图后提供回复建议（针对聊天场景）

### 路径 B：AI 对话

1. 统一时间线，自然语言交互，流式输出
2. 主模型路由到相关子模型，子模型返回结果后主模型汇总
3. 支持上下文记忆（会话级）
4. 长按语音输入
5. 回复建议卡片（快捷回复）
6. 新对话时显示记忆注入区（可折叠）

### 路径 C：手动录入

1. 结构化表单，按模块字段录入
2. CSV 批量导入（财富账单、成长学习记录）
3. CSV 导入向导（字段映射、预览、校验）

---

## 二、Butler 中枢（主页）

1. 5 卡 Bento 布局：1 AI 洞察大卡 + 4 模块快览卡
2. AI 洞察卡展示跨模块叙事（主模型汇总）
3. 模块快览卡展示最新简报标题 + 摘要
4. 简报新鲜度展示（"更新于 X 小时前"，过期则标记）
5. 智能推荐行（基于反馈学习）
6. 数据看板大屏（Web 端）

---

## 三、关系模块 ❤️

1. 联系人 CRUD（姓名、标签、备注、最近互动）
2. 联系人列表（按最近互动/标签/字母排序）
3. 联系人详情（基本信息 + 互动时间线）
4. 互动记录（通话/见面/消息，手动录入）
5. 关系分组（家人/朋友/同事/客户）
6. 聊天记录导入（截图识别 → 结构化存储）
7. 智能回复建议（基于历史聊天风格）
8. 约定管理（与某人的约定，到期提醒）
9. 人情往来记录（送礼/收礼、金额、场合）
10. 关系图谱（Web 端可视化，节点=人，边=关系类型）

---

## 四、成长模块 🎯

1. 目标 CRUD（年度目标/季度目标）
2. 目标进度追踪（百分比 + 关键结果）
3. 学习记录（课程/书籍/技能，手动录入）
4. 成长时间线（按时间倒序展示）
5. 项目看板（看板视图：待办/进行中/已完成）
6. OKR 管理（目标 + 关键结果 + 进度）
7. 简历管理（版本化）
8. 人脉分析（结合关系模块，分析社交网络价值）

---

## 五、财富模块 💰

1. 余额总览（多账户：现金/银行卡/支付宝/微信）
2. 收支记录 CRUD（金额、类别、时间、备注）
3. 月度收支统计（收入/支出/结余）
4. 类别分析（食/行/购/娱等占比）
5. CSV 批量导入账单（支付宝/微信导出格式）
6. 月度账单页（分类汇总 + 趋势）
7. 预算管理（类别预算 + 超支提醒）
8. 自然语言查询（"上个月吃饭花了多少"）

---

## 六、思想模块 🧠

1. 读后感 CRUD（书名、作者、感悟、评分）
2. 思想卡片（灵感速记，标签分类）
3. 思想时间线
4. 标签筛选
5. 给未来的信（定时解锁，未来某日才能阅读）
6. 年度回顾（年末自动汇总全年思想）
7. 双向链接笔记（类似 Roam Research，`[[ ]]` 链接）

---

## 七、健康模块 🏥

1. 健康记录 CRUD（体重/睡眠/运动/心率，手动录入）
2. 健康时间线
3. 基础趋势图（体重/睡眠折线）
4. 体检报告解析（截图识别 → 结构化指标）
5. 健康指标卡（异常项高亮）
6. 综合健康分（多维度加权评分）
7. 设备同步（Apple Health / 小米运动）

---

## 八、记忆系统 🧠

### 8.1 四层记忆架构

1. Layer 1 短记忆：会话级上下文（当前对话）
2. Layer 2 工作记忆：任务态临时存储（当前任务相关）
3. Layer 3 长记忆：跨会话 RAG 检索
4. Layer 4 用户画像：长期稳定的用户特征

### 8.2 RAG 语义记忆

1. 文档切分（按语义段落，重叠窗口）
2. Embedding 向量化（BGE-M3 / OpenAI text-embedding-3-small）
3. 向量检索（sqlite-vec，Top-K 召回）
4. Rerank 重排序
5. 上下文注入（主模型对话时自动注入相关记忆）
6. 语义搜索结果页（展示相似度 + 来源）

### 8.3 反馈学习

1. 6 类反馈信号采集（点赞/点踩/采纳/忽略/修改/分享）
2. 5 条学习规则（调整推荐权重、TTL、简报优先级等）
3. 学习记录页（展示学习效果）
4. 效果边界约束（防止过度学习）

---

## 九、Agent 引擎

### 9.1 多智能体调度

1. 主模型路由（根据用户意图选择子模型）
2. 子模型按需检索回答 + 生成简报（≤2KB，含 push_headline/summary/recommendations）
3. 简报系统（Hive 存储 + generated_at + TTL，覆盖写入）
4. 多层级路由（Level 1-5，复杂度递增）
5. 自适应 TTL（简报有效期动态调整）
6. 子模型自适应调度（按负载/成本择优）

### 9.2 工具系统

1. 工具注册表 ToolRegistry
2. 工具隔离（注册层隔离：子 agent 构造时不注入写工具）
3. ReAct 范式（推理 → 行动 → 观察）
4. 工具清单（各模块 CRUD + RAG + 简报发布，18+ 工具）

### 9.3 错误处理

1. 4 类错误处理：A 未响应 / B 幻觉越权 / C 调用错误 / D 格式错误
2. 熔断器（3 次失败冷却 10min）
3. 监控告警（错误率、延迟、成本）

### 9.4 模型配置

1. ModelRegistry（9 厂商支持：MiniMax/OpenAI/Claude/Gemini/通义/智谱/月之暗面/DeepSeek/本地）
2. 默认 MiniMax M3（原生多模态）
3. 模型配置页（用户可切换）

## 十、导航与全局

1. 胶囊切换器（Butler ↔ 6 模块）
2. 底部 Tab（模块内视图切换）
3. AI 常驻 FAB（任意页面可呼出对话）
4. 模块切换器待处理计数（红点）
5. 设置页（账号/模型配置/数据导出/关于）
6. 启动引导（5 屏 Onboarding）

---

## 十一、跨模块联动

1. EventBus 事件总线（7 类事件）
2. 联系人添加 → 财富分账 / 思想共读
3. 账单超支 → 成长目标影响提示
4. 体检异常 → 思想记录感受 / 财富保险建议
5. 目标达成 → 关系通知 / 健康奖励
6. 读后感完成 → 成长学习记录
7. 7×6 全事件矩阵（完整联动）
8. 吸收器（年度回顾自动汇总全模块）

---

## 十二、Web 端

1. Web 端基础布局（响应式）
2. AI 对话 Web 版
3. 数据查看（只读）
4. 关系图谱可视化
5. 数据看板大屏
6. 自然语言查询界面

---

## 十三、非功能需求

1. 本地优先（数据默认存本地 SQLite）
2. 隐私保护（敏感数据加密，AI 调用脱敏）
3. 流式响应（AI 输出 < 1s 首字）
4. 离线可用（基础 CRUD 无需网络）
5. 端到端加密同步（可选云同步）
6. 成本控制（按真实调用次数估算，详见技术文档）
7. 性能监控（错误率/延迟/成本/Token 用量）
