## 一、输入路径

### 路径 A：浮窗截图（**仅 Android**）

> **平台边界**：仅 Android 实现。iOS 没有 `AccessibilityService` 等价 API、跨 App 浮窗受沙盒限制，详见 [doc/first/04-platform.md §八](file:///d:/others/app/Bulter/doc/first/04-platform.md)。iOS 用户使用"路径 C：手动录入"或"路径 B：AI 对话"。

**交互规范**：

| 用户动作 | Bulter 行为 |
|---|---|
| **单击悬浮球** | **主模型多模态** LLM 看截图 → 选工具 → **直接调工具**写入数据库（关系 / 财富 / 思想 / 健康）→ 顶部短暂通知"已记录到 X 模块" |
| **长按悬浮球**（≥ 500ms） | 浮窗展开语音输入面板 → `speech_to_text` 录音 → 文字实时上屏 → 松手自动 push AI 对话页 + 预填 query |
| **拖拽悬浮球** | 球跟随手指移动 → 松手吸附 |

**实现要点**：

1. 截图走 `AccessibilityService.takeScreenshot()`（**不**用 MediaProjection，避免弹授权框）
2. GPU 缓冲复用：`HardwareBuffer → Bitmap.wrapHardwareBuffer → copy(ARGB_8888)`
3. 三组件架构：`MainActivity`（FlutterActivity 宿主）+ `FloatingBallService`（前台服务空壳保活）+ `ScreenshotAccessibilityService`（无障碍服务做系统级截图）
4. **3s 截图冷却** + 4 个错误码映射（内部错误 / 限流 / 无障碍未授权 / FLAG_SECURE 窗口）
5. **不**弹确认页（用户已在别的 App 里被打断过一次，再弹确认页是二次打断）
6. 失败 → 自动降级到对话页让用户修正

**为什么"单击自动完成"而不是"先确认"**：

- 用户已经在别的 App 里被打断了，再弹个 Bulter 确认页是**二次打断**，违背"被动浏览"原则
- 单击后的 AI 推理在 3s 内完成 + 自动入库 + 顶部轻通知"已记录"——失败回退到对话页让用户修正

**隐私**：截图**永不离开设备**（原图截完即删，仅留 256×256 缩略图作为对话记忆）。

### 路径 B：AI 对话

1. 统一时间线，自然语言交互，流式输出
2. **主模型直接处理**：多模态（看图）+ 工具调用（写库 / 检索 / 简报发布）—— **主模型是核心**
3. 支持上下文记忆（会话级 + 长期 RAG）
4. 长按浮窗 / 浮窗长按启动语音输入
5. 回复建议卡片（快捷回复）
6. 新对话时显示记忆注入区（可折叠）

**架构原则**：

- **主模型** = **多模态 LLM**（默认 MiniMax M3）+ **ToolRegistry 工具调用**：直接处理用户输入、看截图、调工具写库、查 RAG
- **子模型** = 各模块的文本 LLM（关系 / 成长 / 财富 / 思想 / 健康）：仅作为**子模块内部的信息处理者**（生成简报 / 整理本地视图 / 答复合模块查询），**不**负责截图归类（截图归类由主模型多模态完成）
- 跨模块叙事汇总：由主模型直接处理（基于子模型产出的简报）

### 路径 C：手动录入

1. 结构化表单，按模块字段录入
2. CSV 批量导入（财富账单、成长学习记录）
3. CSV 导入向导（字段映射、预览、校验）

## 八、记忆系统 🧠

### 8.1 四层记忆架构

1. Layer 1 短记忆：会话级上下文（当前对话）
2. Layer 2 工作记忆：任务态临时存储（当前任务相关）
3. Layer 3 长记忆：跨会话 RAG 检索
4. Layer 4 用户画像：长期稳定的用户特征

### 8.2 RAG 语义记忆

1. 文档切分（按语义段落，重叠窗口）
2. Embedding 向量化（BGE-M3 / OpenAI text-embedding-3-small）
3. 向量检索（sqlite-vec，Top-K 召回）
4. Rerank 重排序
5. 上下文注入（主模型对话时自动注入相关记忆）
6. 语义搜索结果页（展示相似度 + 来源）

### 8.3 反馈学习

1. 6 类反馈信号采集（点赞/点踩/采纳/忽略/修改/分享）
2. 5 条学习规则（调整推荐权重、TTL、简报优先级等）
3. 学习记录页（展示学习效果）
4. 效果边界约束（防止过度学习）

## 九、Agent 引擎

### 9.1 多智能体调度（**主模型核心 + 子模型辅助**）

**核心架构原则**：

- **主模型**（多模态 LLM，默认 MiniMax M3）= **Agent 引擎的核心**
  - 处理**所有**用户输入（文本 / 截图 / 语音文字）
  - 调 **ToolRegistry** 工具完成实际写库 / 检索
  - 自主判断**是否需要子模型协助**（如要生成简报 / 答复合模块查询）
  - **多模态输入**（`image_url` message）→ 直接看截图 → 选工具 → 调工具

- **子模型**（各模块文本 LLM）= **子模块内部的信息处理者**
  - 仅在**子模块上下文**里被调用（如"生成关系模块今日简报" / "查上月财富分类汇总"）
  - 产出的**简报**（≤2KB，含 `headline` / `summary` / `chips`）写入 Hive / Drift
  - **不**接触截图 / 语音 / 主对话流
  - **不**直接被用户调用（用户只与主模型对话）

**调度流程**：

1. **主模型路由**：判断用户意图（单步可解 vs 需要子模型协作）
2. **单步可解** → 主模型直接调工具 → 返回结果（如截图归类后调 `relationship.add_contact`）
3. **需要子模型** → 主模型调 `invoke_sub_agent(moduleId, query)` → 子模型生成简报 → 主模型汇总
4. **简报发布**：子模型产出 ≤2KB 简报 → 写入 Drift `briefings` 表 + Hive 缓存 → 中枢主页订阅显示

**简报系统**：

1. Hive 存储（冷启动加速）+ Drift 表（持久化）
2. `generated_at` + `TTL`（默认 86400s）
3. **覆盖写入**：每个模块每天只有最新一条简报
4. **自适应 TTL**：根据反馈学习动态调整（活跃模块 TTL 长，闲置模块 TTL 短）

**多层级路由**（Level 1-5，复杂度递增）：

| Level | 描述 | 示例 |
|---|---|---|
| 1 | 主模型直接答 | "今天几号" |
| 2 | 主模型调工具查数据 | "我妈妈生日" |
| 3 | 主模型调子模型生成简报 | "给我今日简报" |
| 4 | 多子模型并行 + 主模型汇总 | "本周我应该关注什么" |
| 5 | 反思 + 决策 + 执行（ReAct 多轮） | "我想离职，帮我分析" |

**自适应调度**：子模型按负载 / 成本择优（高峰期用便宜模型，低峰期用质量模型）。

### 9.2 工具系统

1. **工具注册表** ToolRegistry（18+ 工具：各模块 CRUD + RAG + 简报发布 + 截图处理）
2. **主模型是工具的唯一调用者**——子模型不能直接调工具写库（仅"读"权限）
3. **工具隔离**（注册层隔离）：子 agent 构造时**不**注入写工具（`SubAgentRegistry.register(includeWrite: false)`）
4. **ReAct 范式**（推理 → 行动 → 观察）：主模型可多轮调工具直到任务完成
5. **截图处理工具**：`scene_inference` 是主模型多模态的"内置能力"，**不**是子模型的功能

### 9.3 错误处理

1. 4 类错误处理：A 未响应 / B 幻觉越权 / C 调用错误 / D 格式错误
2. 熔断器（3 次失败冷却 10min）
3. 监控告警（错误率、延迟、成本）

### 9.4 模型配置

1. ModelRegistry（9 厂商支持：MiniMax/OpenAI/Claude/Gemini/通义/智谱/月之暗面/DeepSeek/本地）
2. 默认 MiniMax M3（原生多模态）
3. 模型配置页（用户可切换）

## 十三、非功能需求

1. 本地优先（数据默认存本地 SQLite）
2. 隐私保护（敏感数据加密，AI 调用脱敏）
3. 流式响应（AI 输出 < 1s 首字）
4. 离线可用（基础 CRUD 无需网络）
5. 端到端加密同步（可选云同步）
6. 成本控制（按真实调用次数估算，详见技术文档）
7. 性能监控（错误率/延迟/成本/Token 用量）
