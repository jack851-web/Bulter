# Bulter 总体架构

> Bulter = AI 人生管家。本地优先、越用越懂用户的个人 Agent 工具。
> 目标：把分散的人生数据（关系、成长、财富、思想、健康）汇聚到一个 AI 中枢，由多智能体协作生成洞察与建议。

---

## 一、总体功能

Bulter 围绕"采集 → 理解 → 洞察 → 行动"四步闭环运作：

1. **采集**：通过 3 条输入路径把用户的人生数据接入系统
2. **理解**：多智能体（1 主模型 + N 子模型）分工处理各模块数据
3. **洞察**：每个子模型生成模块简报，主模型汇总生成跨模块叙事
4. **行动**：通过 AI 对话、推送建议、提醒等方式回应用户

---

## 二、三条输入路径

| 路径 | 名称 | 说明 |
|------|------|------|
| A | 浮窗截图 | 系统级 Overlay，任意 App 长按触发截图，AI 推理场景并归类 |
| B | AI 对话 | 统一时间线，自然语言交互，支持语音、多张截图、回复建议 |
| C | 手动录入 | 结构化表单，按模块字段录入，支持 CSV 批量导入 |

---

## 三、六大业务模块

每个模块由一个子模型负责，生成 ≤2KB 的模块简报（ModuleBriefing）。

| 图标 | 模块 | 职责 | 品牌色 |
|------|------|------|--------|
| ❤️ | 关系 | 人脉管理、聊天记录、智能回复、人情往来、关系图谱 | 玫粉 #FF4D8B |
| 🎯 | 成长 | 目标 OKR、项目看板、简历、学习记录 | 深青 #1A3A3A |
| 💰 | 财富 | 余额、账单、预算、CSV 导入、自然语言查询 | 赭金 #E8B94A |
| 🧠 | 思想 | 读后感、给未来的信、年度回顾、双向链接笔记 | 薰衣草 #B8A4ED |
| 🏥 | 健康 | 体检报告解析、指标趋势、综合分、设备同步 | 翠绿 #10B981 |
| 🧠 | 记忆系统 | 4 层记忆架构、RAG 语义检索、用户画像、反馈学习 | — |

> Bulter 中枢（桃色 #FFB084）作为主入口，不归属任何业务模块，负责调度与汇总。

## 四、多智能体架构

> 采用 1 主模型 + N 子模型架构。子模型统一用 API 级轻量模型（DeepSeek-V3 / Qwen-Turbo 等），不使用 7B 本地模型做工具调用。单模块简单问题主模型直接用工具 + RAG 回答，跨模块或需深查才调子模型。

### 4.1 角色分工

| 角色 | 模型 | 职责 |
|------|------|------|
| 主模型 Orchestrator | API 级大模型（≥30B，多模态） | 统一调度、跨模块叙事、写操作、记忆注入 |
| 子模型 Specialist | API 级轻量模型（DeepSeek-V3 / Qwen-Turbo 等） | 本模块按需检索回答 + 生成简报，只读 + RAG |

### 4.2 协作规则

1. 主模型决定何时调用哪个子模型（简单问题直接用 RAG + 工具回答，跨模块才调子模型）
2. 子模型只能读本模块数据 + 调本模块工具 + 发布简报（注册层隔离，不注入写工具）
3. 所有写操作必须经主模型执行 + 用户二次确认
4. **Agent 对话按需检索**，不依赖预生成简报

### 4.3 简报系统（Briefing Store）

> **定位**：首页"被动浏览"功能——用户打开 App 就能看各模块近期摘要，无需主动问 AI。不是 Agent 调度核心。

- 每个子模型定时输出一份 `ModuleBriefing`（≤2KB）：`push_headline` + `summary` + `recommendations`
- 用 `generated_at` + TTL 判断新鲜度，过期下次定时刷新
- 简单覆盖写入，无需 CAS/版本号/状态位（单用户本地无并发）
- **与 Agent 的关系**：简报服务首页浏览；主模型对话回答具体问题靠 RAG + 工具 + `invoke_sub_agent` 按需检索，简报仅作可选快速上下文

### 4.4 调度模式

| 模式 | 触发时机 |
|------|----------|
| 主动-每日 | 每天 23:00 生成日简报 |
| 主动-每周 | 每周日 22:00 生成周简报 |
| 主动-每月 | 每月最后一天 21:00 生成月简报 |
| 主动-每年 | 每年 12/31 20:00 生成年简报 |
| 被动-浮窗 | 截图触发场景推理 |
| 被动-对话 | AI 对话触发相关模块 |
| 被动-主页 | Dashboard 打开触发刷新 |

## 五、导航架构

### 5.1 双层导航

- **顶层**：胶囊切换器（Butler 中枢 ↔ 6 大模块）
- **底层**：底部 Tab（模块内视图切换）+ AI 常驻入口（FAB）

### 5.2 主页（Butler 中枢）

5 卡 Bento 布局：
- 1 张 AI 洞察大卡（跨模块叙事）
- 4 张模块快览卡（关系/财富/思想/健康最新简报）

---

## 六、跨模块联动

通过 EventBus 事件总线实现 7 类事件流转（完整版为 7×6 全事件矩阵）：

| 事件 | 触发 → 接收 |
|------|------------|
| 联系人添加 | 关系 → 财富（分账）/ 思想（共读） |
| 账单超支 | 财富 → 成长（目标影响） |
| 体检异常 | 健康 → 思想（记录感受）/ 财富（保险建议） |
| 目标达成 | 成长 → 关系（通知重要人）/ 健康（奖励） |
| 读后感完成 | 思想 → 成长（学习记录） |
| 约定达成 | 关系 → 财富（人情账） |
| 年度回顾 | 思想 → 全模块（吸收器） |

## 七、设计语言

- 画布底色：#FAF6EE
- 主 CTA：纯黑 #000
- 6 模块各有品牌色（见第三节）
- 图标系统：lucide SVG（统一替代 Emoji）
